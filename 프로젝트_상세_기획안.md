## 1. 프로젝트 개요

**AI-Inv**는 GitHub Actions 스케줄러를 통해 자동으로 글로벌 시장 데이터를 수집하고, Google Sheets에 정제된 형태로 저장한 뒤, Gemini Gems(AI 투자 비서)가 이 데이터를 분석하여 매수/매도 조언 및 시장 브리핑을 제공하는 **1인 자동화 투자 의사결정 시스템**입니다.

```
[GitHub Actions 스케줄러]
         ↓ (2시간 간격, KST 07:00~23:00)
[Python 크롤러 3종 가동]
         ↓
[Google Sheets 데이터 웨어하우스]
         ↓
[Gemini Gems AI 투자 비서 분석]
         ↓
[매수/매도 조언 & 시장 브리핑 출력]
```

## 2. 시스템 아키텍처

### 2-1. 스케줄러 (GitHub Actions)

- **파일 위치:** `.github/workflows/daily_collect.yml`
- **실행 주기:** 평일/주말 무관, KST 기준 오전 7시부터 밤 11시까지 **2시간 간격** 자동 실행
- **중단 시간:** 자정(0시) ~ 오전 6시 (서버 IP 블랙리스트 방어 및 사이트 부하 분산)
- **서버 환경:** GitHub Actions `ubuntu-latest` + Python 3.10
- **타임존 처리:** `pytz` 라이브러리의 `Asia/Seoul` 타임존을 사용하여 GitHub 서버 위치에 무관하게 한국표준시(KST) 기록

### 2-2. 데이터 수집 엔진 (3종 크롤러)

#### (A) Expert_Analytic 크롤러 (`crawler/blog_scraper.py`)

| 수집 방식 | 공식 RSS 피드(`rss.blog.naver.com/ranto28.xml`) 파싱 |

#### (C) News_Calendar 크롤러 (`crawler/finance_scraper.py`)
| 항목 | 내용 |
|---|---|
| 수집 매체 | **5개 채널**, 각 최대 20건씩, **총 최대 100건** |
| 수집 내용 | 기사 제목 + **본문 전체(Full Text)** |
| 본문 파싱 | BeautifulSoup `<p>` 태그 기반, 최대 40,000자 |

**5대 수집 채널:**
1. **CNN Business** — 공식 RSS(`rss.cnn.com`) 기반. 글로벌 경제·증시 속보.
2. **Reuters** — Google News RSS 우회 필터(`news.google.com`) 경유. 국제 금융 보도.
3. **Financial Juice** — 월가 실시간 단문 속보 터미널. JS 렌더링 한계로 헤드라인 위주 수집.
4. **네이버 금융 경제뉴스** — 국내 경제 시황·정책 뉴스.
5. **매일경제(MK) 증권** — 공식 RSS(`mk.co.kr/rss/40300001/`). 국내 증시·산업 뉴스.

### 2-3. 업로더 (`uploader/sheets_api.py`)

#### Snapshot 초기화 전략
과거 데이터가 무한 누적되어 Google Sheets가 느려지는 현상을 방지합니다.
1. 크롤러가 데이터 수집을 완료하면 수집된 배열 유효성 검사(If Valid)
2. **유효할 때만** 해당 탭의 기존 데이터를 전체 클리어(`batch_clear("A2:Z1000")`)
3. 새 데이터를 한 번에 일괄 삽입(`insert_rows`)

#### Bulk Insert 전략
구글 시트의 API Rate Limit(초당 호출 제한) 초과를 방지합니다.  
100건의 데이터를 2차원 배열로 묶어 **단 1번의 API 호출**로 삽입합니다.

## 4. AI 분석 계층 (Gemini Gems)

### 데이터 연동 방식
- Gemini Gems의 지침(Instructions)에 스프레드시트 링크를 하드코딩하여, 사용자가 "오늘 장 어때?" 한마디만 입력해도 AI가 자동으로 시트를 열어 분석을 시작합니다.

### 3인 위원회 토론 구조
```
🐂 황소(Bull)          → 단기 모멘텀 기반 공격적 매수 주장
🐻 불곰(Bear)          → 매크로 리스크 기반 보수적 방어 주장  
🦉 퀀트(Quant)         → RAW 데이터 기반 팩트 체크 및 최종 판정
                              ↓
                    [ 위원회 합의 및 액션 플랜 ]
```

### 출력 형식
1. 📊 시장 현황 브리핑 (핵심 팩트 3가지)
2. 🗣️ 3인 투자 전략 토론
3. 💡 최종 합의 + 오늘의 액션 플랜 3줄
